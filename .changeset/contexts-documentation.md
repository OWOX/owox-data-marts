# Contexts Documentation

Contexts is a new feature that lets Project Admins group Data Marts, Storages, and Destinations by business domain — for example Marketing or Finance — and control which project members can see which resources.

The new documentation covers how to create and manage contexts, how to attach them to resources, how to assign members to contexts, and how the "Selected contexts only" role scope restricts visibility. It also explains when ownership overrides context filtering.
